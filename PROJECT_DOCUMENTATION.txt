1. PROJECT OVERVIEW
================================================================================

CipherSphere is a futuristic, web-based encryption and decryption platform 
built using Flask framework. It provides military-grade cryptographic security
for both text messages and files with a modern cyberpunk-inspired user interface.

PURPOSE:
- Secure encryption and decryption of sensitive data
- User authentication and profile management
- Secure file storage vault system
- File sharing between users with access controls
- Administrative monitoring and management

TARGET USERS:
- Individuals requiring data security
- Organizations needing encrypted file storage
- Security professionals
- System administrators

4. ENCRYPTION ALGORITHMS DETAILS
================================================================================

4.1 AES (Advanced Encryption Standard)
---------------------------------------
Type: Symmetric encryption
Key Size: 256-bit
Mode: CBC (Cipher Block Chaining) with PKCS7 padding
Features:
- Fast encryption for large files
- Industry-standard security
- Efficient memory usage

Use Case: General-purpose file and text encryption

4.2 RSA (Rivest-Shamir-Adleman)
--------------------------------
Type: Asymmetric encryption
Key Size: 4096-bit
Padding: OAEP with SHA-256
Features:
- Public-private key pairs
- Secure key exchange
- Digital signatures capability

Use Case: Small data encryption, key exchange

4.3 Fernet
----------
Type: Symmetric encryption
Based on: AES-128 with HMAC-SHA256
Features:
- Built-in timestamp verification
- Message authentication
- Data integrity checking

Use Case: General-purpose encryption with integrity verification

5. CORE FUNCTIONALITIES
================================================================================

5.1 USER AUTHENTICATION FLOW
-----------------------------
1. Registration:
   - User provides username, full name, email, password
   - Sets security question and answer
   - Password hashed with bcrypt
   - User created in database
   - Activity logged

2. Login:
   - User provides username and password
   - Password verified against hash
   - Session created with Flask-Login
   - Redirects to dashboard or admin panel
   - Login activity logged

3. Password Recovery:
   - User enters username
   - Answers security question
   - If correct, can reset password
   - Password updated in database
   - Recovery activity logged

5.2 ENCRYPTION WORKFLOW
-----------------------
Text Encryption:
1. User enters text content
2. Selects encryption algorithm (AES/RSA/Fernet)
3. Provides or generates encryption key
4. Option to save to vault
5. Encrypted content displayed with download option
6. Activity logged

File Encryption:
1. User uploads file (max 50MB)
2. File type validated against allowed extensions
3. Selects encryption algorithm
4. Provides or generates encryption key
5. File encrypted with metadata preservation
6. Option to save to vault or download immediately
7. Original file deleted from temp storage
8. Activity logged

5.3 DECRYPTION WORKFLOW
-----------------------
Text Decryption:
1. User pastes encrypted text
2. Selects algorithm used for encryption
3. Provides decryption key
4. Text decrypted and displayed
5. Activity logged

File Decryption:
1. User uploads encrypted file
2. Selects algorithm used
3. Provides decryption key
4. File decrypted with metadata extraction
5. Original filename and format restored
6. Decrypted file available for download
7. Activity logged

5.4 VAULT OPERATIONS
--------------------
Upload to Vault:
- File encrypted during upload process
- Stored with unique identifier
- Metadata saved to database
- File size and algorithm tracked

Download from Vault:
- Encrypted file retrieved
- Sent to user with .encrypted extension
- Activity logged

Decrypt from Vault:
- User provides decryption key
- File decrypted in-memory
- Original format restored
- Temporary download link created

Delete from Vault:
- User confirmation required
- File deleted from filesystem
- Database record removed
- Activity logged

5.5 FILE SHARING SYSTEM
-----------------------
Share File:
1. Owner selects file to share
2. Enters recipient's email
3. Sets access level (View/Download/Decrypt)
4. Sets expiration period (days)
5. Share record created in database
6. Both users receive activity logs

Access Shared File:
1. Recipient views shared files
2. Can view/download based on permissions
3. If decrypt permission, can retrieve key
4. Activity tracked

Manage Shares:
1. Owner can view all shares
2. Can revoke access anytime
3. Can extend expiration
4. Auto-expiration enforcement

10.1 PREREQUISITES
------------------

- pip package manager

10.2 INSTALLATION STEPS
-----------------------
1. Clone/Download the project
2. Create virtual environment (recommended):
   python -m venv venv
   venv\Scripts\activate (Windows)
   source venv/bin/activate (Linux/Mac)

3. Install dependencies:
   pip install -r requirements.txt

4. Run the application:
   python app.py

5. Access the application:
   http://localhost:5000
   or
   http://127.0.0.1:5000

10.3 FIRST-TIME SETUP
---------------------
1. Application auto-creates database on first run
2. Admin user auto-created with default credentials
3. Register additional users via web interface
4. Configure admin settings as needed
5. Change admin password immediately

10.4 PRODUCTION DEPLOYMENT
--------------------------
For production use:
1. Use production WSGI server (Gunicorn/uWSGI)
2. Enable HTTPS with SSL certificate
3. Use production database (PostgreSQL/MySQL)
4. Set DEBUG=False
5. Configure environment variables
6. Set up proper logging
7. Implement backup strategy
8. Use reverse proxy (Nginx/Apache)
